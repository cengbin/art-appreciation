# 代码审查工作流

## 代码质量检查

1. 运行 ESLint 检查
```bash
npm run lint
```

2. 检查代码格式
- 缩进是否统一
- 命名是否规范
- 是否有未使用的变量和导入

3. 检查类型安全（TypeScript）
- Props 类型是否定义
- 函数返回值类型是否明确
- 是否滥用 `any` 类型

## 功能检查

4. 测试核心功能
- 页面是否正常渲染
- 交互是否符合预期
- 边界情况是否处理

5. 检查性能
- 是否有不必要的重渲染
- 是否有内存泄漏
- 大列表是否需要虚拟滚动

6. 检查可访问性
- 是否有合适的 alt 文本
- 键盘导航是否正常
- 语义化 HTML 标签

## 代码组织

7. 组件拆分是否合理
- 单一职责原则
- 组件大小适中
- 可复用性

8. 文件结构是否清晰
- 文件命名规范
- 目录组织合理
- 导入路径清晰

## 文档和注释

9. 是否需要添加注释
- 复杂逻辑需要注释
- 公共 API 需要文档
- 避免过度注释

10. README 是否需要更新
- 新功能是否记录
- 使用说明是否完整

## 提交前最终检查

11. 查看所有修改
```bash
git diff
```

12. 确认提交信息规范
- 类型正确（feat/fix/docs 等）
- 描述清晰简洁
- 必要时添加详细说明
